Motivation

The default MongoDB Java driver blocks the calling thread during the entire round trip from the client to the MongoDB server and back. This induces a latency of several to 10's of milliseconds per request. For high transaction applications this latency is difficult to eliminate. The easiest solution to mitigate this overhead is to utilize multiple threads on the client and outbound connections to the servers. Each connection ties up resources on the MongoDB servers that could be utilized for caching data, etc.

The MongoDB Asynchronous Java Driver operates at the core completely asynchronously using a message passing model between the client threads and dedicated I/O threads. To ensure thread safety for the objects passed between the threads almost all BSON and other domain objects are immutable.

Usage

Most users will interact with the driver via the three primary interfaces: {@link com.allanbank.mongodb.Mongo}, {@link com.allanbank.mongodb.MongoDatabase} and {@link com.allanbank.mongodb.MongoCollection}. Interfaces are provided to allow easier mocking in client tests. The implementation of the Mongo interface obtained via the {@link com.allanbank.mongodb.MongoFactory}.

A strongly typed, immutable BSON library is provided via the com.allanbank.mongodb.bson package. It provides the building blocks for BSON documents sent to and retrieved from MongoDB.
